# Testing Guide

每次新功能完成後，在對應章節補充測試案例，確保舊功能沒有 regression。

---

## 環境準備

```bash
# 啟動 backend
cd backend && npm run dev

# 啟動 frontend（另開 terminal）
cd frontend && npm run dev
```

打開 http://localhost:5173，以 Google 帳號登入後開始測試。

---

## 功能一：基本對話

### T1-1 送出訊息並串流回應
1. 新增對話，輸入 `你好` 並送出
2. **預期**：AI 回應逐字串流顯示，完成後傳送按鈕恢復可用

### T1-2 Markdown 渲染
1. 輸入 `請用 markdown 列出三種排序演算法，包含程式碼範例` 並送出
2. **預期**：回應中標題、清單、程式碼區塊正確渲染，無多餘換行

### T1-3 多輪對話
1. 送出 `我叫小明` → 再送出 `你記得我叫什麼嗎?`
2. **預期**：AI 能回答出「小明」（memory cutoff 範圍內）

## 功能二：長期記憶

### T2-1 手動新增記憶
1. 開啟右側「長期記憶」頁籤
2. 輸入框填入 `我是軟體工程師，目前住在台北`，點「新增記憶」
3. **預期**：清單立刻出現新條目（標籤「手動」），不需重整頁面

### T2-2 萃取記憶（基本）
1. 新增對話，送出：
   > `你好，我叫小明，我是台大資工系大三，目前在做一個使用 RAG 技術的 GenAI 專案`
2. AI 回應後點 🧠 **萃取記憶**
3. **預期**：toast 顯示「已新增 N 條記憶」，清單自動刷新，出現：
   - `使用者名叫小明`
   - `使用者就讀台大資工系`
   - `使用者目前在做 RAG / GenAI 專案`

### T2-3 萃取不含 AI 說的話
1. 繼續同一對話，送出：`我想了解計算機視覺`
2. AI 回應後（AI 可能提到 OpenCV、Python 等）點 🧠 萃取
3. **預期**：**不應**新增「使用者喜歡 Python」等 AI 自行推斷的事實

### T2-4 去重複
1. 新增第二個對話，送出：`我叫小明，是台大的學生`
2. AI 回應後點 🧠 萃取
3. **預期**：toast 顯示「沒有新的記憶」或新增數量為 0，不重複儲存語義相同的條目

### T2-5 刪除記憶
1. 在長期記憶清單中，點任一條目右側的 ✕
2. **預期**：條目立即消失，不需重整

### T2-6 Firebase 持久化
1. 確認清單中有數條記憶
2. 重啟 backend（`Ctrl+C` → `npm run dev`），重整前端
3. **預期**：記憶完整保留

## 功能三：對話壓縮

### T3-1 壓縮一般模式對話
1. 新增對話，依序送出：
   > `我最近在學 PyTorch，想做一個影像分類器`

   等回應後再送出：
   > `我打算用 ResNet50 做 transfer learning`

2. 左側對話 hover → 點 ⚡，確認提示視窗後點確定
3. **預期**：
   - 對話替換成一段摘要（包含 PyTorch、ResNet50 等關鍵字）
   - 摘要上方顯示 `⚡ 已壓縮` 標籤
   - 🧠 萃取按鈕消失

### T3-2 壓縮 Discord 模式對話
1. 新增 Discord 模式對話，送出：
   > `我在規劃一個電商網站，前端用 React，後端用 FastAPI`
2. AI 回應後，左側 hover → ⚡ 壓縮
3. **預期**：
   - 摘要正確顯示
   - Discord 樣式訊息上方出現 `⚡ 已壓縮` 標籤
   - hover 工具列不顯示 🧠 按鈕

### T3-3 空對話無法壓縮
1. 新增一個沒有任何訊息的空對話
2. Left sidebar hover
3. **預期**：⚡ 按鈕為灰色禁用，無法點擊

## 功能四：Discord 模式

### T4-1 訊息佇列與批次送出
1. 新增 Discord 模式對話
2. 輸入 `第一句話`，按 Enter（**不**送出，加入佇列）
3. 輸入 `第二句話`，按 Enter
4. 點「全部送出 (2)」
5. **預期**：兩則訊息同時顯示，AI 回應一次性回覆全部

### T4-2 Emoji 反應
1. 在 AI 回應上 hover，點 😊
2. 選擇一個 emoji
3. **預期**：訊息下方出現 emoji 計數，再點同一個 emoji 取消

### T4-3 回覆功能
1. hover AI 訊息，點 ↩
2. 輸入框應顯示回覆對象，送出後訊息上方顯示引用區塊

## 功能五：檔案附件 / 圖片

### T5-1 上傳文字檔
1. 點 📎，選擇一個 `.txt` 或 `.py` 檔案
2. 送出 `幫我解釋這份程式碼`
3. **預期**：AI 能正確描述檔案內容

### T5-2 上傳圖片（Vision 模型）
1. 切換至支援 Vision 的模型（如 `meta-llama/llama-4-scout-17b-16e-instruct`）
2. 點 📎 上傳一張圖片，送出 `這張圖片有什麼?`
3. **預期**：AI 正確描述圖片內容

### T5-3 非 Vision 模型上傳圖片
1. 切換至非 Vision 模型
2. 嘗試上傳圖片
3. **預期**：顯示警告，無法送出

## 功能六：對話管理

### T6-1 新增 / 切換對話
1. 點「+ 新對話」，新增兩個對話並各自送訊息
2. 點左側切換
3. **預期**：各對話內容獨立，互不干擾

### T6-2 重新命名
1. Hover 對話 → 點 ✏️
2. 輸入新名稱，按 Enter
3. **預期**：名稱更新

### T6-3 刪除對話
1. Hover 對話 → 點 🗑️ → 確認
2. **預期**：對話從清單移除，自動切換至其他對話

### T6-4 群組 + 拖曳
1. 點 📂 新增群組，命名為 `測試群組`
2. 拖曳一個對話到群組內
3. **預期**：對話出現在群組下方

## 功能七：AI 自動選擇模型（Auto-routing）

> **前置條件**：右側設定欄 → 開啟「AI 自動選擇模型」開關。

### 背景原理

每次送出訊息前，前端會先呼叫 `POST /api/route`，用 `llama-3.1-8b-instant` 分析你的訊息並選出最適合的模型，再用選出的模型呼叫 `POST /api/chat/stream`。模型選擇結果以小 badge 顯示在每條 AI 回覆旁邊。

### T7-1 簡單問題 → 路由至輕量模型

1. 開啟「AI 自動選擇模型」
2. 新增對話，送出：
   > `你好`
3. **預期**：AI 回覆旁的 badge 顯示 `llama-3.1-8b-instant`

---

1. 送出：
   > `今天天氣怎麼樣?`
2. **預期**：badge 顯示 `llama-3.1-8b-instant`

### T7-2 複雜任務 → 路由至高階模型

1. 新增對話，送出：
   > `請幫我分析以下商業計畫的優缺點，並提供詳細的改善建議：「我想開一家結合 AI 技術的連鎖餐廳，利用 AI 進行點餐、庫存管理與個人化推薦」`
2. **預期**：badge 顯示 `llama-3.3-70b-versatile`

2. **預期**：badge 顯示 `llama-3.3-70b-versatile`

### T7-3 關閉 auto-routing 時 badge 不顯示

1. 關閉「AI 自動選擇模型」
2. 送出任意訊息
3. **預期**：AI 回覆旁**不顯示** badge，使用設定欄中手動選擇的模型

### T7-4 多輪對話各自顯示正確 badge

1. 開啟「AI 自動選擇模型」
2. 送出 `你好` → 送出 `請詳細解釋量子糾纏的原理與實際應用`
3. **預期**：第一條回覆旁顯示 `llama-3.1-8b-instant`，第二條顯示 `llama-3.3-70b-versatile`

### T7-5 無 API Key（使用後端環境變數）

1. 前端設定欄的 Groq API Key 欄位**留空**（後端 `.env` 已設定 `GROQ_API_KEY`）
2. 開啟「AI 自動選擇模型」，送出：
   > `請幫我評估 PostgreSQL 和 MongoDB 的適用情境`
3. **預期**：routing 正常運作，badge 顯示 `llama-3.3-70b-versatile`，不因前端無 key 而失敗

### T7-6 Badge 歷史保留

1. 送出幾條訊息（含簡單與複雜），完成後向上捲動
2. **預期**：每條舊的 AI 回覆旁 badge 仍正確顯示當時使用的模型（存在訊息物件上，不因後續訊息改變）

## 除錯提示

| 問題 | 查看位置 |
|------|---------|
| 萃取失敗 | backend console `[memory/extract]` |
| 壓縮失敗 | backend console `[compress] summary length:` |
| Firebase 連線錯誤 | backend console `[memory/list]` 或 `[memory/add]` |
| 串流中斷 | browser DevTools Network → `/api/chat/stream` |
| Auto-routing 沒有切換模型 | DevTools Network → `POST /api/route`，查看 response `{ modelId, reason }` |
| Auto-routing badge 沒有出現 | 確認訊息物件上有 `model` 欄位：DevTools → Application → localStorage → `ccgpt_conversations` |
